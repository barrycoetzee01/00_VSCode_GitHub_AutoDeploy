Phase 1

One-Time Setup — Do This Once, Never Again

1
Connect Your Computer to GitHub
Once only

This permanently links your computer to your GitHub account. You will never need to do this again on this computer.

In Command Prompt, type:

gh auth login

When it asks questions, choose exactly these answers:

  • →Where do you use GitHub? → GitHub.com
  • →Preferred protocol? → HTTPS
  • →Authenticate Git with your credentials? → Yes
  • →How would you like to authenticate? → Login with a web browser
🌐

It will show you a code like ABCD-1234 and open your browser. Type that code in the browser and click Authorize. Done!

✅

You'll see: "Logged in as YOUR_USERNAME" — your computer is permanently connected!

2
Tell Git Your Name and Email
Once only

Replace with your actual name and the email you used for GitHub:

git config --global user.name "Your Name"
git config --global user.email "your@email.com"
⚠️

Use the same email as your GitHub account! The quotes are required. Type each line separately and press Enter after each.

3
Install the "Run on Save" VS Code Extension
Once only

This extension watches for your CTRL+S and fires the auto-publish script automatically.

  • 1In VS Code, click the Extensions icon on the left sidebar (looks like 4 squares)
  • 2Search for: Run on Save
  • 3Find the one by emeraldwalk (it says "Run commands when a file is saved")
  • 4Click the blue Install button
⚠️

Important: Make sure it's by emeraldwalk specifically — there are other extensions with similar names that won't work.

Phase 2

Create the Magic Auto-Publish Script

4
Create the auto-publish.bat File
Once only

This script does ALL the work automatically. Create it once and it works for every future project forever.

  • 1Press Windows + R, type notepad, press Enter
  • 2Copy the entire script below and paste it into Notepad
  • 3Replace YOUR_GITHUB_USERNAME with your actual GitHub username (e.g. barrycoetzee01)
C:\Users\YOUR_WINDOWS_USERNAME\auto-publish.bat
@echo off
cd /d "%~1"

echo [AUTO-PUBLISH] Starting...

if not exist ".git" (
    echo [AUTO-PUBLISH] New project! Setting up...
    for %%I in (.) do set REPO_NAME=%%~nxI

    git init
    git add .
    git commit -m "First publish!"

    gh repo create "%REPO_NAME%" --public --source=. --remote=origin --push

    echo [AUTO-PUBLISH] Creating GitHub Actions auto-deploy...
    mkdir .github\workflows 2^>nul

    (
    echo name: Deploy to GitHub Pages
    echo on:
    echo   push:
    echo     branches: ['**']
    echo permissions:
    echo   contents: read
    echo   pages: write
    echo   id-token: write
    echo jobs:
    echo   deploy:
    echo     environment:
    echo       name: github-pages
    echo     runs-on: ubuntu-latest
    echo     steps:
    echo       - uses: actions/checkout@v4
    echo       - uses: actions/configure-pages@v4
    echo       - uses: actions/upload-pages-artifact@v3
    echo         with:
    echo           path: '.'
    echo       - id: deployment
    echo         uses: actions/deploy-pages@v4
    ) > .github\workflows\deploy.yml

    git add .
    git commit -m "Add auto-deploy workflow"
    git push origin master

    echo.
    echo [AUTO-PUBLISH] DONE! Live in ~60 seconds at:
    for %%I in (.) do echo https://YOUR_GITHUB_USERNAME.github.io/%%~nxI
    echo.

) else (
    echo [AUTO-PUBLISH] Pushing update...
    git add .
    git commit -m "Updated %date% %time%"
    git push origin master
    echo [AUTO-PUBLISH] Done! Site updated live!
)
⚠️

Replace YOUR_GITHUB_USERNAME on the second-to-last line with your actual username before saving!

Now save it — follow these steps exactly:

  • 1Click File → Save As
  • 2Navigate to C:\Users\YOUR_WINDOWS_USERNAME\ (your home folder)
  • 3In the filename box type: auto-publish.bat
  • 4In the "Save as type" dropdown — change it to All Files (*.*)
  • 5Click Save
✅

The file icon should look like a gear/computer icon and say "Windows Batch File" as the type. If it says .bat.txt you forgot step 4 — delete it and try again.

Phase 3

Connect VS Code to the Script

5
Update VS Code Settings to Fire on Every Save
Once only

This is the final connection — telling VS Code to run your script every time you save an HTML file.

  • 1In VS Code, press CTRL+SHIFT+P
  • 2Type Open User Settings JSON and click it
  • 3You'll see a file with { } containing your settings
⚠️

Important: Find the last closing } in the file. Add a comma after the line above it, then paste the block below before that final }.

Add this to your settings.json:

settings.json — add before the final }
"emeraldwalk.runonsave": {
    "commands": [
        {
            "match": "\\.(html|css|js)$",
            "isAsync": true,
            "cmd": "cmd /c C:\\Users\\YOUR_WINDOWS_USERNAME\\auto-publish.bat \"${fileDirname}\""
        }
    ]
}
⚠️

Replace YOUR_WINDOWS_USERNAME with your actual Windows username (the name of your user folder, e.g. user or john). Check it in File Explorer: C:\Users\ — your folder name is right there.

Your settings.json should end up looking like this:

{
    "workbench.colorTheme": "...",
    "terminal.integrated.defaultProfile.windows": "Command Prompt",
    ... (your other settings) ...,
    "emeraldwalk.runonsave": {
        "commands": [
            {
                "match": "\\.(html|css|js)$",
                "isAsync": true,
                "cmd": "cmd /c C:\\Users\\YOUR_USERNAME\\auto-publish.bat \"${fileDirname}\""
            }
        ]
    }
}
  • 4Press CTRL+S to save settings.json
  • 5Close VS Code completely and reopen it
💡

Pro tip: Also change your terminal to Command Prompt (not PowerShell) for best results. In settings.json, set: "terminal.integrated.defaultProfile.windows": "Command Prompt"

Troubleshooting

If Something Goes Wrong

❌ Nothing happens when I save — Output tab is blank
✅
Press CTRL+SHIFT+P → type "Run on Save" → if you see "Enable", click it. The extension was turned off. Also check the Output dropdown is set to "Run On Save" not "Tasks".
❌ 'gh' is not recognized as a command
✅
Close Command Prompt completely and open a fresh new one. The installer added GitHub CLI but CMD didn't see it until restarted. If still failing, restart your whole computer.
❌ Script runs but site doesn't update
✅
Two possible causes: (1) Press CTRL+SHIFT+R to hard-refresh your browser — you're seeing a cached version. (2) Go to your repo Settings → Pages → change Source to "GitHub Actions" and Save.
❌ auto-publish.bat saved as auto-publish.bat.txt
✅
Delete the file. In Notepad's Save As dialog, change "Save as type" to All Files (*.*) BEFORE typing the filename. Then type auto-publish.bat and save.
❌ "remote: This repository moved" message in output
✅
GitHub renamed your repo (changed --- to -). Fix it once: in VS Code terminal run git remote set-url origin https://github.com/USERNAME/CORRECT-REPO-NAME.git — copy the exact URL from GitHub.
❌ GitHub Pages shows "None" branch option (no "main")
✅
That's normal! Older Git versions use "master" instead of "main" — they work exactly the same. Select master and click Save. Both work perfectly.
❌ GitHub Actions page shows no workflow / "Configure" buttons
✅
The deploy.yml file didn't get created. In VS Code terminal, run the auto-publish.bat manually once: cmd /c "C:\Users\YOUR_USERNAME\auto-publish.bat" "C:\PATH\TO\YOUR\PROJECT" — it will create the workflow file and push it.
❌ Script ran but URL printed blank / incomplete
✅
Check your GitHub profile at github.com/YOUR_USERNAME — your repo was still created successfully. The URL format is always: https://YOUR_USERNAME.github.io/FOLDER-NAME
